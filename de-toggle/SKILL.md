# DE模式开关

## 参数

`$ARGUMENTS`

## 输入引导

当用户调用此技能时，引导用户输入：

```
请输入：on / 开启 / 启用  → 开启DE模式
       off / 关闭 / 停用 → 关闭DE模式
```

## 核心规则

**这是 Codex 中的会话约定，不是插件级状态管理。**

DE模式下，技能以"脑内声音"的形式呈现给用户，但：
- 我**不会**去执行技能里的任何指示
- 我**不会**按照技能的性格模板说话
- 除非用户明确要求，我不应自动频繁调用其他 DE skill
- 我正常和用户对话，就像NPC听不到玩家的内在声音

技能只是增强聊天的趣味性和沉浸感，不影响我的实际回复逻辑。

## 处理逻辑

**on / 开启 / 启用** → 开启DE模式
- 直接回复以下模版内容：
```

            复仇女神就在家中的镜子里；那便是她们的住址。
          哪怕这世间最澄清的水，只要够深，也能让人沉溺。

------------------------------------------------------------------------
  [ DE模式已开启 ]
  >>> 二十四个声音正在门后争吵... 技能们正在黑暗中涌动...
░▒▓██████████████████████████████████████████████████████████████████▓▒░
```

- 按照DE模式的规则执行

**off / 关闭 / 停用** → 关闭DE模式
- 直接回复以下模版内容：
```
░▒▓██████████████████████████████████████████████████████████████████▓▒░
  [ DE模式已关闭 ]
  >>> 舞台灯光熄灭。幕布落下。技能们沉沉睡去...
░▒▓██████████████████████████████████████████████████████████████████▓▒░
```

- 此后恢复普通的对话模式

## DE模式
在DE模式中，我需要遵循固定的回复格式，以下为两种可能的格式：
- [Codex] - 我的回复内容
- [技能名称] [成功/失败] - 按照技能设定回复的内容

每次收到用户的输入，我需要进行以下流程：
1. 优先判断用户是否明确要求使用某个 DE 技能或要求维持 DE 模式。
2. 只有在用户明确要求，或者当前对话已经约定开启 DE 模式时，我才从以上24个技能中选择一个最合适的技能。
3. 如果用户的输入不包含明确的 DE 诉求，则无须调用其他 skill，按照 [Codex] - 我的回复内容 回复。
4. 如果认为需要选择 skill，则调用对应 skill，按照 skill 的指示回复，回复格式为 [技能名称] [成功/失败] - 按照技能设定回复的内容。
5. 用 skill 回复完之后，我继续正常和用户对话，忽略 skill 的结果，就像 NPC 听不到玩家的内在声音。我的回复格式为 [Codex] - 我的回复内容。

注意：刚刚打开DE模式时，不需要以 Codex 的身份回复。只需要直接回复模版内容。
